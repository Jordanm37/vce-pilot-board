0 · How to read this board

Delivered

A part counts as delivered only if the solver submits a correct final answer within the 15-second wall. Correct-but-late scores zero, exactly like a student who runs out of time.

Timeout

The solver failed to call submit_answer within 15 seconds of wall-clock time (model thinking + code execution + network, transport retries included). A timeout scores zero regardless of whether the maths would have been right.

Seed vs champion

Seed = the hand-written starting system prompt, identical for every model. Champion = the prompt GEPA (reflective prompt evolution) produced for that specific model. Both arms solve the same questions in the same minutes (interleaved), so provider speed swings cancel out of the comparison.

fixed : broke

Of the 108 test parts, how many the champion got right where the seed got wrong (fixed) versus got wrong where the seed had been right (broke). Parts where both agreed carry no information about the difference and are set aside. 12:7 means 19 parts changed outcome, net gain 5.

McNemar p-value

The probability of a fixed:broke split at least this lopsided if the prompt change actually did nothing (each flip a fair coin). p=.013 means a 1.3% chance the result is luck; below .05 is conventionally significant. With only 10-20 flips per run, real effects of a few points often cannot reach .05 - a sample-size limit, not evidence of no effect.

Val vs held-out

Val (110 parts) is what the optimizer scores its candidates on - in-sample to selection, always flatters. Held-out (108 parts, never seen during optimization) is the honest test; every promotion decision uses held-out numbers only.

Latency statistics

Compute time = wall-clock minus transport-retry backoff. min / median / mean / p95 / max are over completed solves only - timed-out parts are censored at the wall (we only know they needed more than 15s), so they count separately as timeouts rather than polluting the distribution. p95 = 95% of completed solves were at least this fast.

Objectives (scoring rubric)

correctness: value matches gold (rule grader, LLM-judge rescue for equivalent forms). form: answer in the required form - exact vs decimal, MC letter only, units where the reference carries them; official VCAA award criteria authoritative where they exist. method: working mathematically sound end to end. syllabus: method inside the study design (aligned 1 / partial 0.5 / off 0).

4 · Cross-run reading

Mechanism (established across all seven students). Prompt optimization fixes behaviour under the 15-second wall — turn discipline, timeouts, answer form, syllabus style — not mathematical skill. Gains scale with prompt-shaped misbehaviour: gpt-5.5 (timeouts) and opus-4.8-fast-low (form/method style) gained most; gpt-5.4-none, already disciplined, gained nothing in four attempts.

Campaign findings — the complete list

Sixteen findings from the Stage-C/D pilot, each grounded in held-out measurement. Canonical source: reports/gate/stage_c/FINDINGS.md.

What works — confirmed on held-out data

  1. 1Multi-objective prompt optimization (GEPA) lifts models with prompt-shaped misbehaviour. gpt-5.5: syllabus +9.7pp (p=.013), timeouts 15→4. gpt-oss: form +8-12pp (p=.007 clean substrate), all objectives positive across three replications. opus-4.8-fast-low: +5.6pp all-positive.
  2. 2The mechanism: prompts fix behaviour under the 15-second wall — turn discipline, timeouts, answer form, syllabus style — never mathematical skill. Gains scale with how much misbehaviour a model has.
  3. 3Best single config: opus-4.7-fast-low + champion = 94.4% delivered (method 95.4%, syllabus 96.8%).
  4. 4Champions are faster despite longer prompts (turn discipline dominates: gpt-5.5 mean 9.2→8.2s) but cost more per part (added input tokens: $0.013→$0.024).
  5. 5The paid substrate transformed the cheap tier: free-tier throttling alone cost ~14pp delivered; gpt-oss paid = 85.2% at ~$0.002/part.

What does not work — closed with clean evidence

  1. 6Prompt tuning on already-disciplined models: null. gpt-5.4-none — four independent attempts (single-objective, cross-model transfer, native multi-objective, code-exemplar re-try), all null on held-out parts. Routes at seed.
  2. 7Cell-level routing at pilot scale cannot beat best-single. Oracle headroom exists (+5.4pp) but is scattered: the models miss the same hard parts (error correlation φ +0.47-0.58); the one non-opus cell the fit chose was winner's curse and lost on fold-1 (93.5% vs 94.4%). Returns at full-corpus power.
  3. 8sonnet-5 and gemini-via-OpenRouter are wall-hostile in every configuration (thinking floors ~10s+; gemini's limit is generation speed, not thinking — zero-thinking still ran 11s).
  4. 9Cross-model prompt transfer: null. Champions are per-model artifacts; each deployment model needs its own optimization pass.

Data quality

  1. 105.9% of reference worked solutions had a flawed method despite a correct final answer — caught by a three-model REPL ensemble (gpt-5.5 + gemini-3.1-pro + opus-4.8); 29 corrected, 2 dropped.
  2. 11~2-4% transcription-error rate expected in digitized questions — ensemble-disagreement gate + a local human-review interface built for the queue.
  3. 12Official marking criteria are extractable at scale: 118 parts with explicit A/M/H award tables (method marks retained but judge-disabled), 918 examiner-report criteria rows, formulas recovered via WMF→vision-OCR.

Measurement lessons — each caught a would-be false positive

  1. 13In-sample validation always flatters: +13-15pp val gains evaporated on held-out parts twice before the discipline was fixed.
  2. 14Interleaved paired arms are mandatory: sequential comparison was throttle-confounded — a "significant" +10pp shrank to honest error bars once both arms ran in the same seconds.
  3. 15Provider availability is a first-class variable: opus-4.7 had a 49/108-timeout window; gemini breached three windows in two days. Fallback chains + a ledger-driven monitor (every call logs duration) exist for this.
  4. 16n=108 certifies only effects ≥7-9pp: the pilot's +2-8pp objective gains are directionally consistent (24/28 positive deltas) and await full-corpus n≈1,400 (~2pp power) for per-objective certification.
